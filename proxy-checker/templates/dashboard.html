Schedule Information

• Check Interval: How often to re-check alive proxies only (requires restart)

• Scrape Interval: How often to scrape sources + check all proxies (alive + new) (requires restart)

• Worker Count: Number of concurrent proxy checks (higher = faster but more resource intensive) (requires restart)

• Check Timeout: Maximum time to wait for each proxy response (immediate)

• Test URLs: URLs used to verify proxy connectivity (tried in order) (immediate)

Note: Changes marked "immediate" take effect on next check. "Requires restart" needs app restart.
